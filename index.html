// 01 — المفاهيم الأساسية

HTTP, API & DRF

🌐
HTTP
Hyper Text Transfer Protocol

بروتوكول الاتصال بين الـ Client والـ Server. يُستخدم لنقل البيانات بينهما عبر الإنترنت.
🔌
API
Application Programming Interface

واجهة تسمح للتطبيقات بالتواصل مع بعضها البعض — برنامج يستطيع استخدام خدمات برنامج آخر من خلالها.
⚙️
DRF
Django Rest Framework

Framework قوي يُستخدم لبناء APIs باستخدام Django بشكل سريع واحترافي.
🔄
Request Flow
المستخدم يرسل Request ← الـ API تتحقق منه ← إذا صحيح يتم التنفيذ على Database ← إذا خطأ يُرجع Error.

// 05 — الاستعلامات

Django ORM Lookups

التعبير المعنى
field__gt أكبر من >
field__gte أكبر من أو يساوي ≥
field__lt أصغر من <
field__lte أصغر من أو يساوي ≤
field__exact يساوي تمامًا =
field__contains يحتوي على نص
python # Model
class Product(models.Model):
  name = models.CharField(max_length=100)
  stock = models.IntegerField()

# المنتجات التي stock > 10
Product.objects.filter(stock__gt=10)

# المنتجات المتوفرة (stock > 0)
Product.objects.filter(stock__gt=0)
// SQL المكافئ
SELECT * FROM product
WHERE stock > 10;

// 06 — الـ Views

Class Based Views

🗂️
QuerySet
يحدد البيانات التي سيتم جلبها من قاعدة البيانات.

يتحكم في أي بيانات ستُعرض في الـ API.
queryset = Product.objects.all()
# أو بشرط:
queryset = Product.objects.filter(stock__gt=0)
🔀
Serializer
يحدد كيف ستظهر البيانات في الـ API — يحول Object إلى JSON والعكس، ويتحقق من صحة البيانات.
serializer_class = ProductSerializer
python from rest_framework.generics import ListAPIView
from .models import Product
from .serializers import ProductSerializer

class ProductListView(ListAPIView):
  # أي بيانات سنجلب؟
  queryset = Product.objects.all()

  # كيف ستظهر البيانات؟
  serializer_class = ProductSerializer
// تدفق البيانات في Class Based View
Database → QuerySet → Serializer → View → JSON Response
